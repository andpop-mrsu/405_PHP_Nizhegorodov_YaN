# Лабораторная работа №3: Веб-приложение "Калькулятор" (SPA + REST API)

Одностраничное веб-приложение (Single Page Application), реализованное с использованием PHP-микрофреймворка Slim и базы данных SQLite.

Frontend взаимодействует с backend через REST API, обмениваясь данными в формате JSON.

---

## 📝 Описание

Игра проверяет навыки устного счёта.

При каждом запуске генерируется случайное математическое выражение, которое нужно вычислить.

Каждая игра и её шаги сохраняются в базе данных, а взаимодействие с сервером происходит через HTTP-запросы.

---

## 🎮 Функциональность

* Создание новой игры
* Генерация математических выражений
* Отправка ответа пользователя
* Проверка результата
* Хранение игр и шагов в базе данных
* Просмотр истории игр


---

## ⚙️ Используемые технологии

## 🔌 REST API

Приложение предоставляет следующие маршруты:

### 📥 Получение всех игр

```
GET /games
```

Возвращает список всех игр в формате JSON.

---

### 📥 Получение игры по ID

```
GET /games/{id}
```

Возвращает список шагов конкретной игры.

---

### 📤 Создание новой игры

```
POST /games
```

Пример запроса:

```json
{
  "player_name": "Ярослав"
}
```

Ответ:

```json
{
  "game_id": 1,
  "step_id": 1,
  "expression": "2+3*4"
}
```

---

### 📤 Отправка ответа

```
POST /step/{id}
```

Пример запроса:

```json
{
  "user_answer": 14
}
```

Ответ:

```json
{
  "correct": true,
  "answer": 14
}
```

## 🗄️ База данных

Используется SQLite:

```
db/calculator.sqlite
```

### Таблица `games`

* id — идентификатор игры
* player_name — имя игрока
* game_date — дата создания
* status — статус игры (active, won, lost)

### Таблица `steps`

* id — идентификатор шага
* game_id — ссылка на игру
* expression — выражение
* correct_answer — правильный ответ
* user_answer — ответ пользователя

## 🧩 Архитектура
